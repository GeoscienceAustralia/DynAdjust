218 // From Williams, A. 2018, C++ Concurrency in Action, Manning Publications.
219 //
220 // The standard idiom for preventing copies of a class used to be to declare
221 // the copy constructor and copy assignment operator private and then not
222 // provide an implementation. This would cause a compile error if any code
223 // outside the class in question tried to copy an instance and a link-time
224 // error (due to lack of an implementation) if any of the class' member
225 // functions or friends tried to copy an instance. With C++11, the committee
226 // realized that this was a common idiom but also realized that it's a bit
227 // of a hack. The committee therefore provided a more general mechanism that
228 // can be applied in other cases too: you can declare a function as deleted.
229 //
230 // Disallow use of compiler generated functions. For backward compatibility
231 // with older C++ compilers, keep the "idiom" of private copy constructor
232 // and assignment operator, rather than use delete.
